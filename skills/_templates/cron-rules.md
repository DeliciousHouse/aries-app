# Cron Rules to Live By

## The skill-first checklist

1. **Max 20 lines in a cron prompt.** If it's longer, you're duplicating the skill.
2. **First line: "Read and follow: [skill path]"** Always. No exceptions.
3. **Update the SKILL.md, not the cron.** Only touch the cron if the schedule changes.
4. **Never paste skill steps into the cron.** Reference the file. Always.
5. **One skill per workflow.** Multiple crons can reference the same skill.
6. **Cron = scheduling + context. Skill = workflow + logic.** Never mix them.
7. **Always use isolated sessions.** Never run crons in the main session.
8. **If you change a process, change the skill first.** The crons follow automatically.
