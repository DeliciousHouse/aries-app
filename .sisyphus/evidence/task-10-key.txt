# T10 Evidence: Idempotency Key in Hermes Submission

## Implementation Summary

Added deterministic idempotency key generation to Hermes submission payload and HTTP headers.

### Key Generation

- **Function**: `generateIdempotencyKey(ariesRunId, workflowVersion, tenantId)`
- **Algorithm**: SHA-256 hash of `${ariesRunId}|${workflowVersion}|${tenantId}`
- **Output**: 64-character hex string (sha256)
- **Deterministic**: Same inputs always produce identical key

### Changes Made

1. **backend/marketing/ports/hermes.ts**
   - Added `createHash` import from `node:crypto`
   - Added `generateIdempotencyKey()` helper function
   - Modified `submissionPayload()` to include `idempotency_key` field in all three cases:
     - Resume case (social content weekly)
     - Run case (social content weekly)
     - Fallback case (other workflows)
   - Modified `invoke()` to extract idempotency key from payload and add `Idempotency-Key` HTTP header

2. **tests/hermes-idempotency.test.ts** (new file)
   - Test: Deterministic key generation from aries_run_id + workflow_version + tenant_id
   - Test: Idempotency-Key HTTP header included in submission
   - Test: Different aries_run_id produces different key

### Test Results

All 3 idempotency tests pass:
- ✓ HermesMarketingPort generates deterministic idempotency_key from aries_run_id + workflow_version + tenant_id
- ✓ HermesMarketingPort includes Idempotency-Key HTTP header in submission
- ✓ HermesMarketingPort idempotency_key changes when aries_run_id changes
