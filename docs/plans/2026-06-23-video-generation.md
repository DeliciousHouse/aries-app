## Goal

Close the loop so Aries can generate short-form vertical video (Reels / Stories /
feed video) for the weekly social-content pipeline and publish it to Instagram +
Facebook. The **publish** half (Meta Graph video branches + per-surface
validation) already existed behind `ARIES_VIDEO_PUBLISH_ENABLED`; this work builds
the missing **generation â†’ ingest â†’ synthesize â†’ publish-metadata** half.

Provider decision: **video is generated by Grok / xAI (Grok Imagine)**, invoked by
Hermes. Aries never calls Grok directly and never names the tool â€” Aries sends a
high-level "generate a 9:16 video for this post" instruction and Hermes routes to
its own video tool. Prod publish goes through **Composio** (a new FB/IG video
branch), consistent with the all-Composio prod policy.

## The contract Hermes must honor (load-bearing â€” the whole feature rides on it)

On the **production** stage, Hermes emits video assets in the **same**
`doc.stages.production.primary_output.artifacts.creative_assets[]` array as images,
extending the existing image shape `{assetId, type, path, prompt, placement}`:

| field | type | required | notes |
|---|---|---|---|
| `assetId` | string | yes | stable id (becomes `creative_assets.source_asset_id`) |
| `type` | string | yes | **`"generated_video"`** (image is `"generated_image"`) |
| `media_type` | string | yes | **`"video"`** â€” drives the `creative_assets.media_type` column + all downstream validation |
| `surface` | string | yes | **`"reel"` \| `"story"` \| `"feed"`** â€” drives aspect/duration validation |
| `path` | string | yes | **basename** of an mp4 written into the Hermes image-cache mount (e.g. `"weekly_primary_reel_1.mp4"`); resolved via `<HERMES_IMAGE_CACHE_MOUNT>/<basename>`. Must end `.mp4`/`.mov` |
| `prompt` | string | yes | generation prompt |
| `placement` | number\|string | yes | maps to the `content_package` `post_number` |
| `width` | integer | **yes** | pixels â€” `validateMediaForSurface` fails closed if missing |
| `height` | integer | **yes** | pixels â€” fails closed if missing |
| `duration_seconds` | number | **yes** | seconds â€” reel **3â€“90**, story **â‰¤60**, feed **3â€“60**; fails closed if missing/â‰¤0 |
| `mime` | string | recommended | `"video/mp4"` |
| `aspect_ratio` | string | optional | `"9:16"` for reel/story; Aries derives from width/height if omitted |
| `cover_path` | string | optional | basename of a poster JPG/PNG in the same mount â†’ dashboard `<video poster>` |

Failed renders go in `artifacts.errors[]` (resumability rule â€” never discard
completed clips on a partial failure).

### Two hard requirements

1. **Localize the mp4** into the image-cache mount and report the *basename* â€”
   NOT an expiring Grok CDN URL. Composio rejects media URLs with query
   strings/redirects (signed CDN URLs) for both the IG container and FB
   `file_url`, and localizing reuses the proven image-serving + checksum-idempotency
   + tenant-isolation machinery. (`external_url` remains a documented fallback only.)
2. **`width` / `height` / `duration_seconds` are mandatory.** Every video surface
   fails closed in `meta-media-validation.ts` without them.

### Two-place agreement

The value Aries *publishes off* is the **publish-stage schedule entry**
(`doc.stages.publish.primary_output.schedule[].placement` / `.media_type`, read by
`synthesize-publish-posts.ts`). For a given `post_number`, the `creative_assets[]`
`surface`/`media_type` here and the publish-stage schedule entry **must agree** (a
reel asset needs a `placement=reel` / `media_type=video` schedule entry) or the
post synthesizes as an image and the video is never published.

## Aries-side state (this branch)

Foundation + generation/ingest half are implemented and reviewer-approved; the
Composio publish branch is the remaining Aries slice.

- **Schema** â€” `width_px` / `height_px` / `duration_seconds` (INTEGER, nullable) on
  `creative_assets`, `posts`, `scheduled_posts` (init-db + `migrations/20260623120000_video_media_metadata.sql`).
- **Publish back-half threading** â€” `mediaMetadata` flows
  `scheduled_posts` â†’ scheduled-posts-worker â†’ scheduled-dispatch route â†’
  `dispatchPublish` â†’ `publishToMetaGraph` (direct + Composio seam). Built only for
  a video surface with all three dims present; never fabricated.
- **Ingest** (`ingest-production-assets.ts`) â€” branches on
  `type==='generated_video'` / `media_type==='video'`: writes `media_type='video'`,
  a real `aspect_ratio`, and `width_px`/`height_px`/`duration_seconds`; reuses the
  local-cache read path. Logo-composite is skipped for video.
- **Synthesize** (`synthesize-publish-posts.ts`) â€” copies the linked asset's dims
  onto the `posts` row so the threading above has real values.
- **Outbound request** (`ports/hermes.ts` `VIDEO_EXECUTION_CONTRACT` +
  `workflow-request.ts` per-clip brief) â€” high-level, no tool name; gated on
  `ARIES_VIDEO_PUBLISH_ENABLED` + `videoRenderCount>0`; **byte-identical when off**.
- **Dashboard read** (`workspace-views.ts`) â€” video rows map to `video/mp4` /
  "Generated Video"; the shared `MediaPreview`/`VideoPreview` already render `<video>`.
- **Manual schedule route** â€” threads the three dims (parity with auto-schedule).
