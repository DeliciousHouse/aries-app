# Episode 3 Command / Ops placeholder structures

These are lightweight planning registers for internal engineering execution.
They are not live telemetry.

## Role coverage matrix
Columns:
- Function
- Current owner
- Coverage state
- Evidence
- Next move

Example:
- Runtime page reliability | Jarvis + Roy | Partial | Runtime adapters live; narrow-width QA still needed | Define QA / handoff validator

## Current workload coverage
Columns:
- Work bucket
- Primary owner
- Backup
- Load signal
- Risk

Example:
- Mission Control responsive hardening | Jarvis | Rohan | High | Frontend throughput bottleneck if more Episode 3 scope lands too early

## Capacity gap register
Columns:
- Gap
- Why it matters
- Current impact
- Candidate seat
- Decision needed

## Handoff risk register
Columns:
- Handoff
- Risk
- Current owner
- Failure mode
- Mitigation

## Ownership ambiguity register
Columns:
- Work item
- Why ambiguous
- Interim owner
- Impact
- Resolution path

## Work that keeps landing on Brendan
Columns:
- Work
- Why Brendan still owns it
- Can delegate?
- Candidate owner
- Blocker

## Work Jarvis should probably absorb
Columns:
- Work
- Why it fits Jarvis
- Guardrail
- Current gain
- Open question

## Work that still must stay human

- Why human
