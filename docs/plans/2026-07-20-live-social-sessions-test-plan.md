### NEW CODEPATHS / GUARDS
- Flag-off: every /api/live-sessions route returns real 404 BEFORE tenant/DB (route unit tests)
- Cross-tenant: session id of tenant B under tenant A context → 404 (unit)
- serializeLiveSession(): no payload except reveal carries stream_url/stream_key material — assert over list, detail, calendar, refresh, and error payloads (serializer unit test; structural choke point)
- Partial-unique (tenant, platform) WHERE state='live': second concurrent live insert/transition rejected (requires-infra)
- CAS race: two concurrent transitions, exactly one winner (requires-infra, real Postgres)
- State machine: full legal-transition matrix enumerated; every illegal pair asserted illegal (unit; the table in the plan is the fixture)
- Env parsers: 1/true/yes/on + fallback-on-garbage for every ARIES_LIVE_* knob (unit)
- PATCH reschedule with external_broadcast_id → provider reschedule called (fake provider); without → local only

## Additions from eng-voice review (F18 + fold-ins)
- Abandon sweep pre-cancel guard: fresh platform status read; LIVE/receiving → adopt not cancel; degraded/failed-poll sessions skipped; "cannot cancel active broadcast" error → adopt (fake provider)
- Promo safety: live-promo/live-followup jobs assert publishingRequested:false; created_by clamp limits synthesis to intended posts (mirror reel-clamp tests)
- Reconciler: never cancels an object without the aries:live:<id> provenance marker; double-adoption blocked by (tenant, platform, external_broadcast_id) partial-unique; ambiguous multi-match → nearest-start-wins
- Flag-off drain: master flag off + in-flight rows → scrub/abandon legs still run; no new work claimed
- Bridge dead-node terminal rule: N consecutive failures or past retention horizon → media_type='live' row skipped by sync; sync run not marked partial by dead live nodes
- Idempotency: retry-reuses-key (same UUID across simulated timeout retry → one row)
- Reminders: reschedule regains reminders (startEpoch in key); create-inside-window fires only nearest un-elapsed offset
- IG detection: single empty poll does NOT end session (3-consecutive debounce); error poll never transitions; overlapping windows → nearest-start attribution, one broadcast flips one session
- scheduled→failed on permanent error surfaces attention (never mislabeled "Missed"); failed→scheduled operator reschedule resets attention
- 23505 from partial-unique live index → friendly 409 (distinct code path from CAS 0-row loser)
- DST spring-forward/fall-back wall-time conversion in drawer + reschedule
- Time source: worker predicates use DB now() (fixture proves process-clock skew doesn't shift windows)
- Calendar response with flag off: liveSessions field shape pinned (absent vs empty) for client compat
- Leak test scope: serializeLiveSession payloads + CalendarLiveItem + live_session_events.detail_json + attention_reason + worker log lines — none carry key material; reveal response has Cache-Control: no-store
- Reveal cooldown floor honored (ARIES_LIVE_REVEAL_COOLDOWN_SECONDS)
- Scrub fires from ended (managed, VOD unresolved) after age bound; VOD resolver gives up at 7d and never reaches vod_ready
